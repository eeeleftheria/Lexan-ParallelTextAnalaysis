
# Initializer
Ένα ξεχωριστό πρόγραμμα που δημιουργεί το shared memory segment, τροποποιεί το μέγεθος του ώστε να είναι ίσο με αυτό
του struct sharedObjects, που φέρει όλα τα δεδομένα που θα διαμοιράζονται οι processes, και αρχικοποιεί τα δεδομένα αυτά.
Επίσης, δημιουργεί τις διεργασίες visitors, receptionist, closing.

- initSharedMemory: αρχικοποιεί τα δεδομένα του shared memory

- createRecept: κανει fork εναν receptionist και μέσω exec μεταφέρει τον έλεγχο στο πρόγραμμα του receptionist, περνώντας 
επίσης, το order time, δηλαδή τη μέγιστη δυνατή περίοδο που χρειάζεται για να εξυπηρετήσει έναν επισκέπτη, το όνομα 
του shared memory segment και το όνομα του logging file.

- createVisitor: δημιουργει μέσω fork NUM_OF_VISITORS(ορίζεται ως define) επισκέπτες και μέσω της exec περνάει το restTime,
το όνομα του shared memory segment και το logging file.

- createLoggingFile: ανοίγει ένα αρχείο logging ή το δημιουργεί εαν δεν υπάρχει ήδη, στο οποίο θα καταγράφονται όλες οι
ενέργειες του bar, δηλαδή θα γράφουν σε αυτό όλοι οι visitors & o receptionist τις ενέργειες που πραγματοποιούν.




# Visitor
Όταν έρχεται ένας νέος πελάτης πραγματοποιούνται τα παρακάτω βήματα:

- Εαν το flag isClosing == true, αποχωρεί.

1. Ελέγχει εαν η ουρα αναμονής είναι άδεια και τοποθετείται σε αυτήν. Έαν ναι, τοτε έχει δυο επιλογές: να κάτσει σε κάποιο 
διαθέσιμο τραπέζι ή να παραμείνει στην ουρά αν δεν υπάρχει χώρος.

2. Μέσω της συνάρτησης checkForTable() ελέγχει τη διαθεσιμότητα των τραπεζιών και τοποθετείται σε ένα διαθέσιμο ή επιστρέφεται false
εαν δεν βρέθηκε τέτοιο. Στην περίπτωση αυτή που παραμένει στην ουρά αναμονής, γίνεται suspend στον semaphore στη πρώτη θέση του 
buffer που αναπαριστά την ουρά αναμονής και περιμένει μέχρι να τον ξυπνήσει κάποιος για να κάτσει. 

3. Εαν η checkForTable() επιστρέψει true, δηλαδή ο πελάτης μας τοποθετήθηκε επιτυχώς σε κάποιο τραπέζι, τότε προχωράει στην
παραγγελία του καλώντας την placeOrder(). Η συνάρτηση αυτή δημιουργεί τυχαία μια παραγγελία που συμπεριλαμβάνει τουλάχιστον 1 ποτό
και 1, 2 ή κανενα φαγητό.

4. Αφού πραγματοποιήσει την παραγγελία ξυπνά τον receptionist και ο επισκέπτης κάνει sem_wait στον semaphore που αντιστοιχεί στην καρέκλα
που κάθεται, με σκοπό να περιμένει για την προετοιμασία της παραγγελίας του.

5. Μόλις ειδοποιηθεί από τον receptionist οτι η παραγγελία του είναι έτοιμη (μέσω sem_post στον semaphore της καρέκλας του), ο πελάτης
καλεί την stayInBar() συνάρτηση. Η συνάρτηση αυτή υπολογίζει ένα τυχαίο διάστημα για το οποίο ο επισκέπτης θα παραμείνει στο bar μεταξύ
[0.7 * restTime, restTime] και κάνει sleep για την περίοδο αυτή, αφού πρώτα έχει απελευθερώσει το mutex. Όταν, έρθει η ώρα να αποχωρήσει
αφαιρείται απο το τραπέζι και την καρέκλα και μεταβάλλει τα απαραίτητα δεδομένα (occupied seats etc).

6. Σε περίπτωση που ο πελάτης προς αποχώρηση είναι ο τελευταίος του συγκεκριμένου τραπεζιού πρέπει να μαρκάρει το τραπέζι ως διαθεσιμο
(isFull = false) και να ξυπνησει τους επομενους πελατες που περιμενουν στην ουρα αναμονης. Η λειτουργία αυτή υλοποιείται μέσω της 
wakeUpWaitingVisitors(). Εαν ο αριθμός πελατών που περιμένουν είναι μικρότερος ή ισος του 4, τότε ο επισκέπτης τους ξυπνάει όλους. 
Διαφορετικά, ξυπνάει μόνο τους 4 πρώτους.

# Receptionist
Ο receptionist είναι υπεύθυνος για τη διαχειριση των παραγγελιών και το κλείσιμο του bar που συμπεριλαμβάνει και τηναπελευθέρωση της μνήμης. 
Όταν, κάποιος πελάτης υποβάλλει την παραγγελία του,  ξυπνάει τον receptionist (sem_post), ώστε να αρχίσει να την προετοιμάζει. Η επεξεργασία της παραγγελίας πραγματοποιείται με την εξής σειρά:
1. Εαν υπάρχουν παραγγελίες στον circular buffer ordersOrder με τα orders, παίρνει την πρώτη απο αυτές.

2. Την επεξεργάζεται και ενημερώνει τους αντίστοιχους counters των στατιστικών μέσω της updateStats()

3. Κάνει sleep για ένα τυχαίο διάστημα μεταξύ [0.5 * torder, torder] που υπολογίζεται, αλλά πρώτα απελευθερώνει τον semaphore mutex
προκειμένου να μην περιμένουν όλοι οι υπόλοιποι το διάστημα αυτό. 

4. Αφού προετοιμάσει την παραγγελία, την αφαιρεί απο την ουρά παραγγελιών, εντοπίζει το τραπέζι και την καρέκλα στην οποία κάθεται ο πελάτης 
της παραγγελίας μέσω των findTableOfVisitor() και findChairOfVisitor() και τον ξυπνάει με sem_post() στον semaphore της καρέκλας του.

- Η διαδικασία closing του μαγαζιού περιγράφεται παρακάτω.


# Closing
To process αυτό δημιουργείται απο τον Initializer και εκτελείται με όρισμα τον χρόνο που θέλουμε το bar να παραμείνει ανοιχτό.
Το διάστημα αυτό, θέλουμε το πρόγραμμα να κάνει sleep και αφού έρθει η ώρα για το κλείσιμο του μαγαζιού, θέτει το flag isClosing
σε true. Επιπλέον, πρέπει να ξυπνήσει τον receptionist προκειμένου να τον ενημερώσει για την μεταβολή αυτή.  
Οι visitors ελέγχουν πριν φύγουν εαν το closing flag ειναι true. Ο τελευταίος πελάτης πρέπει να ξυπνήσει τον receptionist ώστε να
μπορέσει να προχωρήσει ομαλά στην διαδικασία του κλεισίματος. O receptionist κάθε φορά ελέγχει εαν το flag αυτό ειναι true, κοιτάει 
εαν όλα τα τραπέζια είναι άδεια μέσω της checkIfBarIsEmpty() και ταυτόχρονα εαν η ουρά αναμονής είναι κενή. Όταν ισχύουν και τα δυο, 
καλεί την closeBar() η οποία εκτυπώνει τα τελικά στατιστικά του μαγαζιού, καταστρέφει τους semaphores και καταστρέφει το shared memory 
segment. Ο receptionist είναι ο μόνος που αφαιρεί την κοινή μνήμη με την shm_unlink(). Ωστόσο, κάθε διεργασία οφείλει να κάνει munmap().
